# 테스트/품질 관리

## 1) 현재 테스트 자산

백엔드 테스트 파일:

- `app/tests/auth_apis/test_signup_api.py`
- `app/tests/auth_apis/test_login_api.py`
- `app/tests/auth_apis/test_token_api.py`
- `app/tests/auth_apis/test_social_auth_service.py`
- `app/tests/user_apis/test_invitation_apis.py`
- `app/tests/user_apis/test_document_drugs_apis.py`
- `app/tests/user_apis/test_ocr_parser_service.py`
- `app/tests/user_apis/test_mfds_service.py`

핵심 검증 범위:

- 인증 플로우(회원가입/로그인/토큰)
- 초대코드 연동 플로우
- 문서 추출약 조회/확정
- 문서 삭제 시 관련 복약데이터 비활성화
- OCR 파서 노이즈 제거/스케줄 파싱

## 2) 실행 방법

백엔드 테스트:

- `uv run pytest app/tests`

정적 분석(백엔드):

- `uv run ruff check app`
- `uv run mypy app`

프론트엔드:

- 현재 `npm run lint` 스크립트 없음
- 최소 빌드 검증: `cd frontend && npm run build`

## 3) 품질 게이트 제안

- PR 필수:
- `pytest` 통과
- `ruff` 통과
- `mypy` 통과(점진 적용 가능)
- 프론트 `npm run build` 통과

- 배포 전 필수 시나리오:
- 로그인/토큰 갱신
- 문서 업로드 -> OCR -> 추출약 확정
- 문서 삭제 후 복약체크 반영 제거
- 보호자/복약자 모드 전환

## 4) 보강 필요 테스트

- DRUG-011/013(DUR 경고 매핑) 자동 테스트
- OCR 재시도 실패/복구 시나리오 테스트
- 프론트 E2E(모드 전환, 알림센터, 복약체크 동작) 테스트
