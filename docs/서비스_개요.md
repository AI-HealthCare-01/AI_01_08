# 서비스 개요

## 1) 서비스 목표

AI Health는 다음 문제를 해결하는 것을 목표로 합니다.

- 처방/복약 문서를 OCR로 구조화해 복약 정보를 자동 정리
- 복약자와 보호자가 동일한 데이터 기준으로 복약 상태를 확인
- 미복용/지연 복약에 대한 알림 및 리마인드 제공
- 의료/복약 관련 안내를 AI 가이드/채팅으로 보조

## 2) 주요 사용자

- 복약자(PATIENT): 본인 문서 업로드, 복약체크, 일정 확인, 건강프로필 관리
- 보호자(CAREGIVER/GUARDIAN): 연동된 복약자 관리, 복약상태 모니터링, 리마인드 발송
- 관리자(ADMIN): 대시보드 기반 운영 지표 확인

## 3) 핵심 가치

- 데이터 일관성: 문서(OCR) 변경이 복약체크/스케줄에 반영되어야 함
- 접근 통제: 환자-보호자 연동 관계 기반 권한 검증
- 운영 가능성: Docker/Nginx/FastAPI 기반으로 배포 및 모니터링 가능

## 4) 서비스 범위

### In Scope

- 계정: 로그인/회원가입/토큰 갱신/소셜 로그인 시작
- 연동: 초대코드 생성/연동/목록/해제
- 문서: 업로드/목록/삭제/OCR 상태/재시도/추출약 조회 및 확정
- 복약: 복약체크/취소/건너뛰기/현황/이행률
- 일정: 병원 일정 CRUD
- 알림: 목록/읽음/삭제/설정/수동 리마인드
- AI: 가이드 생성/조회/재생성, 채팅 세션/메시지/피드백

### Out of Scope (현재 코드 기준)

- 프론트엔드 단독 정식 배포 자동화(예: Vercel 전용 설정 파일)
- 프론트엔드 lint 파이프라인(현재 `npm run lint` 스크립트 없음)
