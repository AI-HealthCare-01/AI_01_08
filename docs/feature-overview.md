# 기능 개요

## 서비스 요약
진료 기록(처방전/문서)을 기반으로 복약 관리, AI 가이드, 환자-보호자 연동, 알림까지 한 흐름으로 제공하는 서비스입니다.

## 역할별 기능

| 역할 | 주요 기능 |
|---|---|
| PATIENT(복약자) | 이메일 가입/로그인, 건강 프로필 관리, 문서 업로드(OCR), 약 정보 확정, AI 가이드/챗봇, 복약 체크, 병원 일정, 알림 확인 |
| CAREGIVER(보호자) | 초대코드 연동, 연동 환자 복약/일정/문서/가이드 확인, 미복약 확인, 수동 리마인드 전송 |
| ADMIN | 운영 대시보드 지표 조회(사용자/가이드/OCR/챗봇/에러) |

## 핵심 기능

### 1) 인증/권한
- 이메일 회원가입/로그인과 역할 기반 접근 제어를 제공합니다.
- 역할에 따라 화면/데이터 접근 범위가 분리됩니다.

### 2) 환자-보호자 연동
- 환자는 초대코드를 생성하고 만료시간 내 공유할 수 있습니다.
- 보호자는 코드 입력으로 연동하며, 연동 해제도 가능합니다.
- 설정에서 다크모드를 토글하고 사용자 설정으로 저장합니다.

### 3) 문서 OCR 및 약 정보 확정
- 처방전 업로드 후 OCR이 비동기로 처리됩니다.
- 상태는 `queued`, `processing`, `success`, `failed`로 관리됩니다.
- 추출 약 정보(약명/용량/횟수/기간)를 수정 후 확정할 수 있습니다.
- 확정 정보는 복약 스케줄과 가이드/챗봇 컨텍스트에 반영됩니다.

### 4) AI 가이드 및 챗봇
- 가이드 생성/재생성은 `ai_tasks` 큐 기반 비동기 워커로 처리됩니다.
- 챗봇 응답은 `chat_tasks` 큐 기반 비동기 처리와 세션 컨텍스트를 사용합니다.
- 환자/보호자 모드별 응답 톤을 적용하고 응급 키워드 안내를 제공합니다.

### 5) 복약 체크, 병원 일정, 알림
- 복약 체크는 개별/시간대 일괄 처리, 건너뛰기, 취소를 지원합니다.
- 복약 미니달력으로 일자별 상태를 확인할 수 있습니다.
- 병원 일정은 등록/수정/삭제/조회(CRUD)를 제공합니다.
- 알림센터에서 조회/읽음/삭제를 지원하며 자동 알림과 수동 리마인드를 제공합니다.
- 알림 처리는 Redis 큐 + 워커 구조로 동작합니다.

### 6) 홈/운영 대시보드
- 홈 화면에서 역할별 요약(KPI, 일정, 복약 상태, 최근 알림)을 제공합니다.
- 관리자 화면에서 OCR/가이드/챗봇 운영 지표를 확인할 수 있습니다.

## 웹 시연 흐름 요약
1. 랜딩 페이지 확인 후 로그인
2. 환자 초대코드 생성, 보호자 연동, 설정(다크모드) 확인
3. 처방전 업로드 및 OCR 비동기 처리 상태 확인
4. OCR 약 정보 수정/확정
5. AI 가이드 생성 및 결과 확인
6. 챗봇 상담(환자/보호자 관점)
7. 복약 체크와 달력 반영 확인
8. 병원 일정, 바코드 약 검색, 알림센터, 관리자 대시보드 확인
