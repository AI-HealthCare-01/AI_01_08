# 요구사항 정의서 (원본 반영본)

## 문서 메타

- 원본 기준: 사용자 제공 요구사항 정의서 (`최종 상태 2026-02-25` 표)
- 문서화 기준일: 2026-03-19
- 목적: 팀 공통 기준 요구사항을 ID 단위로 명확히 고정

## 표기 규칙

- 이 문서는 **원본 요구사항을 그대로 우선 반영**합니다.
- 원본 표에는 `REQ-AUTH-006`, `REQ-AUTH-007`이 서로 다른 의미로 **중복 사용**되어 있습니다.
- 혼동 방지를 위해 본 문서에서만 아래처럼 별칭을 병기합니다.
- `REQ-AUTH-006A`: 역할 선택/변경
- `REQ-AUTH-006B`: 토큰 갱신
- `REQ-AUTH-007A`: 권한(Authorization) 접근제어
- `REQ-AUTH-007B`: 비밀번호 재설정

## 1) 인증/권한 (AUTH)

| ID | 요구사항 | 핵심 내용 | 우선순위 |
|---|---|---|---|
| REQ-AUTH-001 | 이메일 회원가입 | 이메일/비밀번호/닉네임/이름/휴대폰/생년월일/성별, 이메일/휴대폰 인증, 닉네임 중복 확인 | 높음 |
| REQ-AUTH-002 | 이메일 로그인/토큰 발급 | Access/Refresh 발급, 이메일 미존재/비밀번호 불일치 구분 처리 | 높음 |
| REQ-AUTH-003 | 카카오 소셜 로그인/회원가입 | OAuth2, 기존 사용자 중복 생성 방지, 필수 동의 항목 반영 | 높음 |
| REQ-AUTH-004 | 네이버 소셜 로그인/회원가입 | OAuth2, 기존 사용자 중복 생성 방지, 필수 동의 항목 반영 | 높음 |
| REQ-AUTH-005 | 로그아웃/토큰 무효화 | Refresh Token 무효화(블랙리스트/삭제) 정책 포함 | 높음 |
| REQ-AUTH-006A | 역할(Role) 선택/변경 | PATIENT/CAREGIVER 역할 정책 적용 | 높음 |
| REQ-AUTH-007A | 권한(Authorization) 접근제어 | PATIENT 본인 데이터만, CAREGIVER 연동 환자만, 위반 시 403 | 높음 |
| REQ-AUTH-006B | 토큰 갱신 | Access 만료 시 Refresh 재발급, 저장 정책 정의 | 중간 |
| REQ-AUTH-007B | 비밀번호 재설정 | 이메일 인증(또는 링크) 기반 재설정 | 낮음 |
| REQ-AUTH-008 | 회원탈퇴 | 연동/문서/로그 데이터 처리 정책(비활성화/지연삭제) | 낮음 |
| REQ-AUTH-009 | 약관동의 | 가입 시 개인정보/이용약관 필수 동의 | 높음 |
