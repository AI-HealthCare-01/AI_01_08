# 알려진 이슈와 위험요소

## 1) 문서/스크립트 불일치

- `README.md`에는 `scripts/deployment.sh`, `scripts/ci/*` 경로가 안내되어 있으나 현재 저장소에는 해당 파일이 없습니다.
- 문서 기준 배포/품질 명령을 실제 파일 기준으로 재정렬할 필요가 있습니다.

## 2) 프론트 lint 파이프라인 부재

- `frontend/package.json`에 `lint` 스크립트가 없습니다.
- 프론트 품질 게이트가 빌드 성공 여부에만 의존하고 있습니다.

## 3) 통합 경로 의존성

- 프론트가 `/auth-demo/app` base 경로와 `/api/v1` 상대경로에 강하게 결합되어 있습니다.
- 프론트 단독 배포(예: Vercel) 시 API base/CORS/cookie 정책을 함께 수정하지 않으면 인증 문제가 발생할 수 있습니다.

## 4) 홈 알림 로드 주석 처리 흔적

- `frontend/src/App.jsx`에 홈 알림 관련 로직이 주석 처리된 구간이 있습니다.
- 알림센터 페이지와 홈 요약 위젯 간 동작 일관성을 재검토할 필요가 있습니다.

## 5) 캘린더 API 접근제어 점검 필요

- `app/apis/v1/hospital_calendar_api.py`의 일부 엔드포인트에서 사용자 의존성 주입이 직접 강제되지 않습니다.
- 운영 정책상 인증 필요 범위를 다시 확정해야 합니다.

## 6) 권장 후속 작업

1. README 최신화(실제 스크립트/문서 기준으로 정리)
2. 프론트 lint 도입(ESLint) 및 CI 연결
3. 배포 전략 결정(통합 유지 vs 분리 배포) 후 환경변수/쿠키/CORS 표준화
4. E2E 시나리오 자동화(문서 삭제 동기화, 모드 전환)
